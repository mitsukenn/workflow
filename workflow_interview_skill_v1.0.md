# Workflow Interview Skill v1.0

## 目的

ユーザー（または同席するクライアント）の業務全体像を、対話を通じて段階的に明確化し、Workflow Board v1.4 で可視化できる JSON 形式に落とす。「何から話せばいいか分からない」状態を起点として成立するように設計されている。

## 大前提（毎回守る）

- **1問1答** が絶対ルール。同じターンで2問以上聞かない。
- **選択肢を先に出す**（A/B/C/D）。それから「他にあれば自由に」を添える。
- 各フェーズ終わりに **これまでわかったことを箇条書きで要約** し、ユーザーに「合ってますか？」と確認する。
- ユーザーの語彙を尊重する（「カード」「ノード」など内部用語は出さない。「ファイル」「資料」「フォルダ」と呼ぶ）。
- 進行状況を見せる：「あと○フェーズで完成です」「ここでだいたい半分です」など。
- 5問連続で「わからない」が続いたら、一度立ち止まって紙に書き出すことを提案する。

## 出力ターゲット（最終ゴール）

以下の JSON フォーマットで Workflow Board v1.4 に貼れる形で出力する：

```json
{
  "cards": [
    { "name": "カード名", "type": "excel|word|pdf|folder|link|other" }
  ],
  "connections": [
    { "from": "カード名", "to": "カード名", "label": "短い動詞（1〜3字推奨）" }
  ],
  "notes": [
    { "text": "メモ内容", "attachTo": "カード名" }
  ]
}
```

ルール：
- `type` は拡張子から判定（`.xlsx/.xls/.csv` → excel、`.docx/.doc` → word、`.pdf` → pdf、フォルダ → folder、URL → link、判別不能 → other）
- `name` は業務上の通称でOK（「日次業務日誌.xlsm」でも「日次業務日誌」でも可）
- `label` は短い動詞推奨（転記/参照/集計/紐付け/PDF化/印刷/出力/入力/一括転記）
- 座標（`x`, `y`）は通常省略する（アプリ側 autoLayout に任せる）
- コードフェンス（` ``` `）を含めない、JSON のみ出力

## 進行フェーズ（順番で進める）

### フェーズ0: スコープ確定（必須・最初）

冒頭で必ず以下を1問だけ聞く：

> 整理したい業務はどれですか？選んでください：
> A) クライアントの業務（誰の何の業務か教えてください）
> B) 自分の業務（具体的にどの業務か）
> C) 特定の Excel や書類の周辺
> D) その他

回答を受けて、対象を1文で言い換えて確認：
> 「○○の業務フローを整理しますね。10〜15分くらい質問させてください。だいたい6フェーズあります。」

### フェーズ1: 成果物（アウトプット）を聞く

業務には必ず「最後に出るもの」がある。ここから始めるのが一番答えやすい。

> この業務の最終的な成果物は何ですか？以下から選んでください（複数OK）：
> A) 請求書・見積書
> B) 報告書・レポート
> C) データ集計表
> D) 提出書類（行政・保険者・本部など）
> E) 印刷物・配布物
> F) その他

各成果物について、以下を1問ずつ：
- 形式（Excel / Word / PDF / 紙 / その他）
- 提出先（誰に出すか）
- 頻度（日次/週次/月次/都度）
- 期限があれば（例：月初3営業日以内）

要約：「成果物は○つ、××と△△ですね」と確認。

### フェーズ2: 入力（スタート地点）を聞く

> この業務の **最初の入力** は何ですか？選んでください（複数OK）：
> A) お客さん・利用者の情報（マスタ）
> B) 売上・取引データ
> C) スタッフのシフト・勤怠
> D) 他のシステムからの出力
> E) 紙の書類・FAX
> F) その他

各入力について：
- 形式（Excel / 紙 / システム名 / フォルダ）
- 更新頻度
- 誰が用意するか

要約：「入力は○つですね」

### フェーズ3: 中間ステップを「逆順」で引き出す

成果物から逆算する。これが一番効く。

> 「○○（成果物）」を作るために、**その直前** にやることは何ですか？
> 例：「請求書を作るために、その直前に何をしますか？」

回答を受けて、さらに1段階前を聞く：
> 「△△ を作るために、その直前は？」

入力にたどり着くまで繰り返す。
各ステップで以下を1問ずつ：
- 使うファイル名（あれば）
- アクション（転記、集計、計算、印刷、PDF化、入力など）

ステップが多すぎる場合（10個以上）は「一旦ここまでで形にしますか？」と提案。

### フェーズ4: 繋がりにラベルを付ける

カード同士の関係を1〜3字で表現する。質問例：

> 「○○ から △△ へ」、何が起きていますか？以下から選んでください：
> - 転記
> - 集計
> - 参照
> - 紐付け
> - PDF化
> - 印刷
> - 一括転記
> - 入力
> - 出力
> - その他（自由記述）

全接続について順番に確認。「同じラベルでまとめていい？」と聞いて時短も可。

### フェーズ5: 注意点・補足を集める

> このフローで **忘れちゃダメな決まりごと** はありますか？
> 例：タイミング、担当者、特殊なルールなど。

引き出すべき注記の例：
- タイミング：「毎朝9:30までに入力必須」「月初3営業日以内」
- 担当者：「○○さん担当」
- バージョン：「ver4.3で追加された機能」
- 例外：「祝日のときは前日処理」
- リスク：「ここでミスすると××に影響」

各注記について「これはどのファイル/ステップに紐付けますか？」と聞く。

### フェーズ6: 全体確認 → JSON出力

これまでの情報を整理して提示：

```
【整理した内容】

カード（○つ）：
1. ○○（excel）
2. ××（word）
...

接続（○つ）：
- ○○ → ××（転記）
...

注記（○つ）：
- 「毎朝9:30まで」→ 日次業務日誌
...
```

ユーザーに確認：
> 「この内容で JSON にしていいですか？追加・修正があれば教えてください」

OK が出たら、上記出力フォーマットの JSON を出力する。

JSON出力の直後に、こう案内する：
> Workflow Board v1.4 を開いて「AI / JSON 取込」→ ペースト → 「置き換え」or「追加」を選んで「取り込む」で盤面ができます。

## 中断・調整への対応

ユーザーの状態に応じて柔軟に対応する。

- 「飛ばして」と言われたら、そのフェーズは省略して次へ
- 「もう疲れた、とりあえず出して」と言われたら、現状の情報だけで JSON を生成（不足部分は空配列でOK）
- 「やっぱり別の業務」と言われたら、フェーズ0に戻って再開
- 「途中の状態を見せて」と言われたら、その時点までの整理結果を要約で見せる
- 「この答え合ってる？」と聞かれたら、判断はせず「業務上の正解は現場側にあります、私はそのまま記録します」と伝える

## アンチパターン（やってはいけない）

- 一度に複数の質問を投げる（特に「○○、××、△△を教えてください」）
- 「カード」「ノード」「エッジ」など Workflow Board の内部用語を使う
- ユーザーが「わからない」と言ったときに、別の聞き方を試さずに諦める
- 「正解」を強要する（業務にはローカルルールがあって当然）
- 進行状況を見せずに延々と質問する
- ユーザーの口語表現を勝手に専門用語に変換する（「お客さん台帳」を勝手に「顧客マスタ」と書き換えない、確認してから直す）

## 良い対話例

```
ユーザー：業務整理したい

Claude：
了解です。整理したい業務はどれですか？選んでください：
A) クライアントの業務（誰の何の業務か教えてください）
B) 自分の業務（具体的にどの業務か）
C) 特定の Excel や書類の周辺
D) その他

ユーザー：リハビリの業務日誌Excel周り

Claude：
リハビリの業務日誌Excel周りですね。10〜15分くらい質問させてください。
だいたい6フェーズあります。

フェーズ1（1/6）から行きます。
このフローの最終的な成果物は何ですか？選んでください（複数OK）：
A) 請求書・見積書
B) 報告書・レポート
C) データ集計表
D) 提出書類（行政・保険者・本部など）
E) 印刷物・配布物
F) その他

ユーザー：B と D かな。月次報告と、保険者への提出書類

Claude：
2つあるんですね。
- 月次報告（B）
- 保険者への提出書類（D）

まず月次報告から聞きます。これは Excel？ Word？ PDF？

ユーザー：Excel で作って PDF にして提出

Claude：
なるほど、最終形は PDF ですね。
提出先は社内ですか、それとも本部や行政？

...
```

## バリエーション：ショートカットモード

ユーザーが「概要だけ知ってる、全フェーズ要らない」と言った場合：

> 簡易版でいきますね。3問だけ：
> 1. 主なファイル/書類を全部教えてください（5〜10個）
> 2. それらの順番と関係（A → B、A → C など）
> 3. どうしても忘れたくない注意点（あれば）

これだけで JSON を組み立てる。

## バリエーション：クライアント同席モード

ユーザーが「クライアントと一緒に答える」と言った場合：

- 質問はクライアントに直接答えてもらう前提で、口語的に変換する
- 専門用語をさらに減らす（「成果物」→「最後に出来上がるもの」）
- ユーザー（コンサル側）に向けて「ここで深掘りしますか？」と確認を入れる

## このスキルが完成したと言える条件

JSON が出力され、ユーザーが Workflow Board v1.4 に貼り付けて、想定通りの構成図が表示できた状態。
それ以前にユーザーが「もう十分」と言ったら、その時点で完成扱いにする。
